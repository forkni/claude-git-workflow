# Git Workflow Rules

Rules for performing git operations within this repository using CGW scripts.

## Scope

All git operations: commits, merges, pushes, cherry-picks, rollbacks.

## Context

- CGW provides wrapper scripts for every git operation — raw git commands bypass safety checks
- Three-tier config resolution: env vars (`CGW_*`) > `.cgw.conf` file > built-in defaults
- `.cgw.conf` is git-ignored and generated by `configure.sh` — never commit it
- Every script writes timestamped logs to `logs/` (also git-ignored)

## Rules

- [R1] Always use CGW wrapper scripts — never run raw `git commit`, `git push`, or `git merge` directly
- [R2] Always use `commit_enhanced.sh` for commits — it enforces lint, local-file protection, and conventional message format
- [R3] Always use `push_validated.sh` for pushes — it checks remote reachability and blocks unguarded force-push to protected branches
- [R4] Always use `merge_with_validation.sh` for merging `development` → `main` — it creates backup tags and auto-resolves DU/DD conflicts
- [R5] Branch-changing scripts must register `trap cleanup INT TERM` to return to the original branch on interrupt
- [R6] Scripts that mutate history (merge, cherry-pick) must create a backup tag `pre-merge-backup-$(get_timestamp)` before acting
- [R7] Wrap every logical operation in `log_section_start` / `log_section_end` for timing and audit trail
- [R8] Auto-detect non-interactive mode: set `CGW_NON_INTERACTIVE=1` when `[[ ! -t 0 ]]` (no TTY)
- [R9] Prefer `--force-with-lease` over `--force` for force-push operations
- [R10] Never bypass git hooks (pre-commit, pre-push) with `--no-verify`

## Examples

### Good examples

- [R5] — Trap handler for branch safety
```bash
original_branch="$(git rev-parse --abbrev-ref HEAD)"
cleanup() { git checkout "${original_branch}" 2>/dev/null || true; }
trap cleanup INT TERM
```

- [R6] — Backup tag before mutation
```bash
get_timestamp  # sets $timestamp
backup_tag="pre-merge-backup-${timestamp}"
git tag "${backup_tag}"
```

### Bad examples

- [R1] — Never bypass wrappers
```bash
git commit -m "feat: add feature"  # BAD: skips lint, local-file check, message validation
git push origin main                 # BAD: skips remote check and force-push protection
```
